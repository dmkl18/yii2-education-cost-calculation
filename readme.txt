Пример определения стоимости обучения

Наиболее важная ачсть приложения - расчет стоимости обучения - /speciality/calculate-cost

Расчет стоимости производится на странице /speciality/calculate-cost после того, как пользователь зарегистрируется
Есть две роли - админ и зарегистрированный пользователь

Админ может заполнять неограниченное кол-во заявок, а также может просматривать список всех заявок

Зарегистрированный пользователь имеет возможность заполнить только одну заявку
	Также имеет возможность сменить пароль
	Перейдя к себе на страничку пользователь может посмотреть заполненную им заявку на поступление
	
После заполнения заявки, а также после регистрации на электронный адрес пользователя приходят соответствующие письма с подтверждением

***************************************************************
Таблицы в бд
***************************************************************

users - таблица пользователей
speciality - варианты специальностей
variantSp - конкретный профиль специальности (связь с таблицей speciality - один ко многим, нельзя удалить специальность, пока есть хотя бы один профиль данной специальности)
speciality - в целом список предметов, изучаемых в учебном заведении
specSubject - список предметов конкретной специальности (связь с таблицей speciality один ко многим, с таблицей variantSp - один ко многим, удаление в обоих случаях каскадное)
educationVariant - вариант образования
studyOption - тип обучения (полное или сокращенное)
technology - технология обучения
studyCost - стоимость обучения (сязана с таблицами educationVariant и technology , удаление каскадное)
application - заявка пользователя (зависит от:
	variantSp - нельзя удалить специальность, пока на нее есть хотя бы одна заявка,
	studyOption, educationVariant, technology - при удалении ставится в NULL,
	users - при удалеии пользователя каскадное удаление заявки)
	
Базовые пользователи:
	- админ - topor (пароль topor)
	- пользователи:
		- user1 - (пароль user1)
		- user2 - (пароль user2)
